методика работы Gitflow
___________________________

1) создать рипозиторий на GitHub и склонировать его на компьютер.
2) создать новую ветку "develop" от главной ветки (которая main или master).
3) создать от ветки "develop"  feature/-ветки и мержить в develop, когда они будут готовы.
4) создать ветку release/[версия] от develop.
5) после этого, фичи уже не создаются, только исправляются баги.
6) когда ветка release котова, она мержится в main и в develop и удаляется.
7) если на ветке main обнаруживается ошибка, он неё создаётся ветка hotfix/[название ошибки].
8) когда работа над hotfix/ веткой заканчивается, её нужно мержить в main и develop.


чтобы замержить одну ветку в другую через консоль, нужно сначала переключиться на ту ветку, на которую ты хочишь мержить,
а потом уже написать git merge и ту ветку, которую нужно мержить.

чтобы добавить изменения на определённой ветке в GitHub локально, нужно прописать команду git pull origin + название ветки.
